# CogniStruct — Product Specification Document

## 1. Product Overview

**Name:** CogniStruct  
**Type:** AI-powered learning platform for school students  
**Target Users:** Students in Class 6–10 (ages 11–16)  
**Platform:** Web application (React + Vite SPA, deployed on Vercel)  
**Backend:** Vercel serverless function (`/api/gemini`) + Supabase (PostgreSQL + Auth)  
**AI Engine:** Google Gemini 2.5 Flash Lite (server-side only via API key)

---

## 2. Purpose

CogniStruct helps school students understand their syllabus chapters deeply through AI-generated mind maps, flashcards, quizzes, active recall, and a doubt-solving chat. It personalises learning using the student's grade, hobbies, and chapter progress.

---

## 3. Authentication

### 3.1 Sign Up
- Fields: Full Name, Grade (Class 6–10), Email, Password (min 6 chars)
- Validates full name for inappropriate language before account creation
- Creates a profile row in `public.profiles` via Supabase trigger
- Supports Google OAuth sign-in

### 3.2 Sign In
- Email + password login via Supabase Auth
- Google OAuth sign-in
- All login errors return identical message: "Invalid email or password." (prevents user enumeration)

### 3.3 Session
- JWT session token stored by Supabase client
- Token attached as `Authorization: Bearer <token>` on every `/api/gemini` call
- Server verifies token via `supabase.auth.getUser(token)` before processing any request

## 4. Core Features

### 4.1 Chapter Management
- Students can create chapters by entering a topic and selecting their class level
- AI generates a structured chapter with 5–8 concepts, each with title, description, estimated minutes, and prerequisites
- Chapters are stored in `user_data` table in Supabase per user
- Concepts have statuses: LOCKED, NOT_STARTED, IN_PROGRESS, MASTERED
- Mastery level tracked 0–100 per concept

### 4.2 Whole Chapter Explanation
- Student selects length (Short / Standard / Long) and depth (Basic / Intermediate / Advanced)
- AI generates a full explanation of the chapter with LaTeX math, bold key terms, bullet points
- Explanation saved to `chapter_explanations` table (max 5 per user per chapter — oldest auto-deleted)
- History of previous explanations browsable within the feature

### 4.3 Mind Map
- AI generates a hierarchical mind map for any chapter
- Config options: Complexity (Basic / Advanced) and Detail (Brief / Detailed)
- Visual interactive mind map rendered in the browser
- Nodes have types: CONCEPT, EXAMPLE, FACT
- Mind maps saved to `user_data`

### 4.4 Active Recall
- AI generates a conceptual question for each concept in the chapter
- Student types their answer, then reveals the correct answer
- Helps reinforce memory through self-testing

### 4.5 Flashcards
- AI generates 5 flashcards per chapter (front: question/term, back: answer/explanation)
- Student flips through cards and rates mastery: AGAIN / HARD / GOOD / EASY
- Spaced repetition: `nextReviewDate` calculated based on mastery rating
- Flashcards saved to `user_data`

### 4.6 Doubt Solver (AI Chat)
- Conversational AI chat for asking subject doubts
- Context-aware: knows student's grade, chapter, and concept progress
- Subject-specific behaviour: maths (step-by-step), physics (formula + example), chemistry (everyday analogies), biology (body/nature examples), history (who/what/when/why), geography (real places), civics (Indian government examples), economics (market examples)
- Last 20 messages kept in history per session
- Inappropriate/vulgar messages in any language (English, Hindi, Gujarati) are rejected with a polite redirect
- Chat history saved to `chat_history` table

### 4.7 Revision Mode
- Reviews all concepts marked IN_PROGRESS or MASTERED
- AI-assisted revision prompts

### 4.8 Quiz
- **Types:** MCQ (4 options), FIB (fill in the blank), MATCH (3 pairs), or MIXED
- **Questions:** 5 per quiz session, generated by AI for the chapter's concepts
- **Scoring:** Points awarded per correct answer, saved to `quiz_results`, aggregated into `profiles.total_points`
- **Shuffle:** Options shuffled with unbiased Fisher-Yates algorithm
- **Completion:** `onComplete` fires after final question with score; points written to Supabase
- Quiz type selectable before starting

### 4.9 Leaderboard
- Shows top 50 users by `total_points` (descending)
- Displays: rank, avatar, name, grade, points
- Current user's rank highlighted
- Only users with points > 0 shown
- Medal icons for ranks 1, 2, 3

### 4.10 Community
- Browse other students' public profiles
- Visible fields: name, grade, avatar, hobbies (no email or admin status)
- Send friend requests with optional reason message
- Accept / decline incoming friend requests
- Friend request statuses: PENDING, ACCEPTED, DECLINED
- Push notifications sent via OneSignal when a friend request is received

### 4.11 Profile
- Editable fields: Full Name, Grade, Hobbies & Interests
- Email displayed but not editable (managed by Supabase Auth)
- Avatar upload: click or drag-and-drop, max 5 MB, formats: JPG/PNG/WebP/GIF
- Avatar stored in Supabase Storage `avatars` bucket
- Total quiz points displayed as a badge
- Name and hobbies validated: inappropriate/vulgar words blocked in all languages

### 4.12 Admin / Teacher Dashboard
- Only visible to users where `profiles.is_admin = true`
- Sidebar "Teacher View" button hidden from non-admin users
- Admin can view platform-wide data

## 5. API — `/api/gemini` Serverless Function

All AI operations go through a single Vercel serverless endpoint.

### 5.1 Authentication
- Every request must include `Authorization: Bearer <supabase_jwt>`
- Server verifies JWT with Supabase before processing
- Returns `401 Unauthorized` if token missing or invalid

### 5.2 Rate Limiting
- 20 requests per minute per authenticated user ID
- Returns `429 Too Many Requests` when exceeded

### 5.3 Actions

| Action | Description |
|--------|-------------|
| `explain` | Generate chapter explanation (length + depth configurable) |
| `generate-chapter` | Generate chapter structure with concepts from a topic |
| `generate-mindmap` | Generate hierarchical mind map JSON |
| `generate-question` | Generate single active recall question for a concept |
| `generate-flashcards` | Generate 5 flashcards for a chapter |
| `generate-revision` | Generate revision content |
| `generate-quiz` | Generate quiz questions (MCQ/FIB/MATCH/MIXED) |
| `doubt` | Conversational doubt-solver with subject-aware context |
| `send-notification` | Send OneSignal push notification to a user (UUID validated) |

### 5.4 Security
- CORS: only allows requests from configured origin domains (no wildcard)
- All string inputs sanitised and length-capped
- `targetUserId` validated as UUID v4 before forwarding to OneSignal
- User-controlled data never placed in AI system instruction (prompt injection prevention)
- 30-second AbortController timeout on all AI fetch calls
- `parseInt` always uses radix 10

## 6. Database Schema (Supabase / PostgreSQL)

### `public.profiles`
| Column | Type | Notes |
|--------|------|-------|
| id | uuid | FK → auth.users |
| full_name | text | |
| email | text | synced from auth |
| grade | text | Class 6–10 |
| avatar_url | text | Supabase Storage URL |
| hobbies | text | |
| total_points | integer | default 0, updated by trigger |
| is_admin | boolean | default false |

### `public.quiz_results`
| Column | Type | Notes |
|--------|------|-------|
| id | uuid | PK |
| user_id | uuid | FK → auth.users |
| chapter_id | text | |
| chapter_title | text | |
| quiz_type | text | MCQ/FIB/MATCH/MIXED |
| score | integer | correct answers |
| total | integer | total questions |
| points_earned | integer | |
| created_at | timestamptz | |

### `public.chapter_explanations`
| Column | Type | Notes |
|--------|------|-------|
| id | uuid | PK |
| user_id | uuid | FK → auth.users |
| chapter_id | text | |
| chapter_title | text | |
| content | text | AI-generated explanation |
| length | text | SHORT/STANDARD/LONG |
| depth | text | BASIC/INTERMEDIATE/ADVANCED |
| created_at | timestamptz | |

Max 5 explanations per user per chapter (enforced by trigger).

### `public.user_data`
Stores chapters, mind maps, flashcards, and active recall data as JSON blobs per user.

### `public.chat_history`
Stores doubt-solver conversation history per user.

### Row Level Security
- All tables have RLS enabled
- Users can only read/write their own rows (`auth.uid() = user_id`)
- Profiles readable by all authenticated users (peer visibility for leaderboard/community)
- `is_admin` and `email` never returned in peer profile queries

## 7. Notifications (OneSignal)

- Push notifications sent when friend requests received
- OneSignal initialised with app ID from env var
- `allowLocalhostAsSecureOrigin` only enabled in development builds
- External user ID set to Supabase user ID after login

## 8. Themes

Three themes supported:
- **Light** — default white/slate palette
- **Dark** — slate-900 backgrounds
- **Reading** — warm sepia tones for reduced eye strain

## 9. Non-Functional Requirements

| Requirement | Detail |
|-------------|--------|
| Performance | AI responses within 30s (AbortController timeout) |
| Security | JWT auth on all API calls, RLS on all DB tables, CSP header, no API keys in frontend |
| Availability | Leaderboard shows empty state (not infinite spinner) on network error |
| Content Safety | Profanity filter on all user text inputs (English, Hindi, Gujarati) |
| Accessibility | Keyboard navigable, ARIA labels on interactive elements |
| Responsiveness | Mobile-first, works on screens 320px and above |
